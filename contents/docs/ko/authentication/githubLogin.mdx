## 1. GitHub Developer Settings에서 OAuth 애플리케이션 생성 방법

1. [GitHub Developer Settings](https://github.com/settings/developers)로 이동합니다.
2. `New OAuth App` 버튼을 클릭합니다.

<Image
  src="/images/mdx/github_user_auth_0.png"
  alt="github_user_auth_0"
  width={1024}
  height={768}
  caption="github - user auth"
/>

3. 애플리케이션 이름, 홈페이지 URL, 승인된 리디렉션 URI를 입력합니다.
  - Homepage URL: `http://localhost:3000`
  - 승인된 리디렉션 URI: `https://your-supabase-url/auth/v1/callback`
  - 'Device Flow' 은 선택하지 않은 상태로 둡니다.
4. 새 OAuth 자격 증명 복사하여 Supabase 대시보드에 입력합니다.
  - 클라이언트 ID를 복사하여 저장합니다.
  - 새 클라이언트 비밀번호 생성을 클릭합니다.
  - 클라이언트 비밀번호를 복사하여 저장합니다.

## 2. Google OAuth 설정
1. [Supabase](https://supabase.io) 프로젝트 대시보드에서 `Authentication`- `Providers` 으로 이동합니다.
2. `Providers` 섹션에서 `GitHub`을 선택합니다.

<Image
  src="/images/mdx/github_user_auth_1.png"
  alt="github_user_auth_1"
  width={1024}
  height={768}
  caption="github - user auth"
/>

3. GitHub Developer Settings에서 OAuth 애플리케이션에 생성된 클라이언트 ID와 비밀 키를 Supabase에 입력합니다.
