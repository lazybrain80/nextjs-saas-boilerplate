## 1. How to create an OAuth 2.0 client ID in Google Cloud Console
1. access [Google Cloud Console](https://console.cloud.google.com/) and create a project or select an existing project.
2. go to `APIs and services` > `User credentials`.

<Image
  src='/images/mdx/google_user_auth_0.png'
  alt='google_user_auth_0'
  width={512}
  height={384}
  caption='google api - user auth'
/>

3. Click the `Create user credentials` button and select `OAuth client ID`.

<Image
  src='/images/mdx/google_user_auth_1.png'
  alt='google_user_auth_1'
  width={1024}
  height={768}
  caption='google api - user auth'
/>

4. select `Application type` as `Web application` and enter the required information.
5. add `https://your-supabase-url/auth/v1/callback` to the `Authorised redirect URI`.

<Image
  src='/images/mdx/google_user_auth_2.png'
  alt='google_user_auth_2'
  width={1024}
  height={768}
  caption='google api - user auth'
/>

6. copy the generated client ID and client secret key and enter them into the Supabase dashboard.


## 2. Set up Google OAuth
1. go to the `Authentication` tab in the [Supabase](https://supabase.io) project dashboard.
2. in the `Providers` section, select `Google`.

<Image
  src='/images/mdx/google_user_auth_3.png'
  alt='google_user_auth_3'
  width={1024}
  height={768}
  caption='google api - user auth'
/>

3. create an OAuth 2.0 client ID in Google Cloud Console, and enter the created client ID and secret key into Supabase.
